# 🔌 اتصال به Supabase (نسخهٔ اصلی)

پروژهٔ تو: `https://wsklewekwsrukadqmybd.supabase.co`
فایل `.env` از قبل با آدرس و کلید تو ساخته شده. فقط باید دیتابیس را راه بیندازی.

---

## گام ۱: ساخت جدول‌ها و امنیت
1. وارد داشبورد Supabase شو → منوی **SQL Editor**.
2. کل فایل `supabase_setup.sql` را کپی کن و **Run** بزن.
   - همهٔ جدول‌ها، RLS، باکت‌های Storage و سیاست‌ها ساخته می‌شوند.
   - در پایین پیام می‌بینی که می‌گوید هنوز کاربر یاسین ساخته نشده — این طبیعی است، در گام بعد می‌سازیم.

## گام ۲: ساخت مربی پیش‌فرض «یاسین بیات»
چون Supabase حساب کاربری را در سیستم احراز هویت نگه می‌دارد، اول باید کاربر را بسازی:

1. برو **Authentication → Users → Add user**.
2. این مقادیر را وارد کن:
   - **Email:** `yasin@coachdesk.app`
   - **Password:** یک رمز دلخواه (مثلاً `Yasin@1234`)
   - تیک **Auto Confirm User** را بزن (تا نیازی به تأیید ایمیل نباشد).
3. **Create user** را بزن.
4. حالا دوباره برو **SQL Editor** و فقط این تکه را اجرا کن (یا کل فایل را دوباره Run بزن):
   ```sql
   do $$
   declare uid uuid;
   begin
     select id into uid from auth.users where email = 'yasin@coachdesk.app' limit 1;
     insert into public.profiles (id, full_name, role)
     values (uid, 'یاسین بیات', 'coach')
     on conflict (id) do update set full_name = 'یاسین بیات', role = 'coach';
   end $$;
   ```
5. تمام ✅ حالا با ایمیل `yasin@coachdesk.app` و رمزی که گذاشتی می‌توانی به‌عنوان مربی وارد شوی.

## گام ۳: خاموش کردن تأیید ایمیل (اختیاری ولی توصیه‌شده)
تا وقتی مربی برای ورزشجوها حساب می‌سازد، نیازی به تأیید ایمیل نباشد:
- **Authentication → Providers → Email** → گزینهٔ **Confirm email** را خاموش کن.

## گام ۴: حذف خودکار استوری‌های منقضی (اختیاری)
برای اینکه استوری‌های قدیمی از سرور هم پاک شوند و حافظه اشغال نکنند:
- **Database → Extensions** → افزونهٔ **pg_cron** را فعال کن.
- بعد در SQL Editor اجرا کن:
  ```sql
  select cron.schedule('purge-stories','*/30 * * * *',
    $$ delete from public.stories where expires_at < now() $$);
  ```

## ⚠️ نکتهٔ امنیتی مهم
کلید `anon` تو در گفتگو فاش شد. این کلید برای مرورگر ساخته شده و امنیت واقعی‌ات به **RLS** بستگی دارد (که این اسکریپت کامل تنظیمش می‌کند)، پس فاجعه نیست. ولی برای احتیاط:
- از **Settings → API** می‌توانی هر وقت خواستی کلیدها را **Reset/Rotate** کنی.
- کلید **service_role** را هرگز در فایل‌های فرانت‌اند یا گیت نگذار — فقط در سرور/Edge Function.

## فایل `.env`
از قبل ساخته شده و این دو مقدار را دارد:
```
VITE_SUPABASE_URL=https://wsklewekwsrukadqmybd.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJI...  (کلید تو)
```
> این فایل در `.gitignore` هست و در گیت آپلود نمی‌شود. هنگام دیپلوی روی Vercel، همین دو مقدار را در **Settings → Environment Variables** وارد کن.

## مرحلهٔ بعد
الان اپ هنوز به نسخهٔ محلی (`src/db.js`) وصل است. برای اینکه واقعاً از Supabase بخواند/بنویسد، باید یک لایهٔ اتصال (`src/supabase.js`) بنویسیم که همان توابع `db` را با کوئری‌های واقعی Supabase جایگزین کند. این کار حجم دارد؛ هر وقت آماده بودی بگو تا قدم‌به‌قدم انجامش دهیم.
