# 🚀 آنلاین کردن روی گیت‌هاب + Vercel

پروژه از قبل به‌صورت ریپوی گیت آماده شده (اولین commit زده شده) و به Supabase تو وصل است.
فقط باید روی گیت‌هاب بگذاری و به Vercel وصل کنی.

---

## گام ۱ — ساخت ریپو در گیت‌هاب
1. برو [github.com/new](https://github.com/new)
2. یک نام بگذار، مثلاً `coachdesk`
3. **هیچ گزینه‌ای را تیک نزن** (نه README، نه gitignore) — چون پروژه از قبل این‌ها را دارد.
4. **Create repository** را بزن.
5. گیت‌هاب یک آدرس نشانت می‌دهد مثل:
   `https://github.com/USERNAME/coachdesk.git`

## گام ۲ — بارگذاری پروژه روی گیت‌هاب
پوشهٔ `coach-local` را روی کامپیوترت باز کن و در ترمینال داخل همان پوشه این‌ها را بزن
(به‌جای URL، آدرس ریپوی خودت را بگذار):

```bash
git remote add origin https://github.com/USERNAME/coachdesk.git
git branch -M main
git push -u origin main
```

> اگر گیت ازت نام کاربری/رمز خواست، به‌جای رمز باید **Personal Access Token** بدهی
> (github.com → Settings → Developer settings → Personal access tokens → Generate).
> یا ساده‌تر: از **GitHub Desktop** استفاده کن و پوشه را drag کن.

## گام ۳ — اتصال به Vercel
1. برو [vercel.com](https://vercel.com) و با همان حساب گیت‌هاب وارد شو.
2. **Add New → Project**
3. ریپوی `coachdesk` را از لیست انتخاب و **Import** کن.
4. Vercel خودش تشخیص می‌دهد که Vite است. **قبل از Deploy**، بخش
   **Environment Variables** را باز کن و این دو را اضافه کن:

   | Name | Value |
   |------|-------|
   | `VITE_SUPABASE_URL` | `https://wsklewekwsrukadqmybd.supabase.co` |
   | `VITE_SUPABASE_ANON_KEY` | (کلید anon تو — از فایل `.env` کپی کن) |

5. **Deploy** را بزن. یک لینک مثل `coachdesk-xxx.vercel.app` می‌گیری.

## گام ۴ — تنظیم Supabase برای دامنهٔ جدید
تا ورود و ثبت‌نام روی دامنهٔ Vercel کار کند:
1. در Supabase → **Authentication → URL Configuration**
2. در **Site URL** و **Redirect URLs**، آدرس Vercel خود را اضافه کن
   (`https://coachdesk-xxx.vercel.app`).

## گام ۵ — ورود
- روی گوشی لینک Vercel را باز کن.
- با حساب مربی وارد شو:
  - ایمیل: `yasin@coachdesk.app`
  - رمز: `Yasin@1234`  (اگر عوض کرده‌ای، همان)

## به‌روزرسانی بعدی
هر تغییری که بدهی، فقط کافی است:
```bash
git add -A
git commit -m "توضیح تغییر"
git push
```
Vercel خودکار نسخهٔ جدید را می‌سازد.

## ⚠️ امنیت
- `.env` در گیت آپلود نمی‌شود (در `.gitignore` است). کلید فقط در Vercel به‌عنوان env قرار می‌گیرد.
- کلید `anon` برای مرورگر امن است؛ امنیت واقعی از RLS می‌آید که در `supabase_setup.sql` تنظیم شده.
- چون کلیدت قبلاً در گفتگو دیده شد، اگر خواستی از Supabase → Settings → API می‌توانی reset کنی (بعدش env را در Vercel و `.env` به‌روز کن).
